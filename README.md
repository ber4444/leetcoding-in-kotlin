This repo contains solutions to Cracking the Coding Interview, 6th edition (2015, around the same time LeetCode and Interviewing.io both came into being)

I also included some other goodies and will include some of the Beyond Cracking the Coding Interview (2025, two years after GPT-4 that started solving LeetCode Hard with some success... note that even in 2026, current models are still not producing production-level LeetCode submissions)

For historical context, CtCi was first published in 2008, when GitHub just launched. She was one of the first voices to successfully argue that companies should stop asking riddles and start testing actual coding and data structure knowledge.

_Study plan:_

Prioritize: arrays/strings, hash maps, binary search, two pointers/sliding window, trees, graphs (BFS/DFS), heaps/top‑k, intervals, and “classic mediums.”

Practice in “CoderPad mode”: no IDE autocompletion, type clean Kotlin, then manually test 3–5 cases out loud.

_Resources:_

LeetCode (free tier is fine) for daily repetitions

https://neetcode.io/ for explanations when you’re stuck or your solution is messy; it also has structured DS&A courses if you want a guided track.

Interviewing.io / https://www.tryexponent.com/ = start in week ~16–18. Until you’ve done at least 10 mocks — don’t waste time on real interviews.

For system design: https://github.com/weeeBox/mobile-system-design/ and the book "Mobile system design interview"
